What is this? The worksheet lists every moment in the Game 1 script — all 381 rows. Each row has a pre-filled time estimate for how long it’ll be in the final episode. The estimates are computer-generated guesses. Your job is to scan through and fix the ones that feel wrong. You know this script and this gameplay better than anyone.

Understanding the Defaults

The est_basis column tells you why the default is what it is:

"buffer default 4s"
A 30-second window around a gameplay moment. Default assumes you’ll use ~4 seconds of the key action. Increase for big moments, decrease for quick flashes.
"40% of 69s"
A longer gameplay sequence with defined start/end times. Default uses 40% of raw footage. Increase for featured battles, decrease if there’s dead time.
"50w @ 2.5wps"
An interview segment. The number before “w” is the word count, 2.5 words per second is the assumed speaking pace. Adjust based on your actual delivery.
"image default"
An image or graphic on screen. Default is 3 seconds. Increase for complex images, decrease for quick visual gags.
"needs collection"
Footage that hasn’t been captured yet. Default is 3 seconds as a placeholder. Set to 0 if you plan to cut it.
"group header"
A structural label, not actual content. Always 0 seconds. Ignore these rows.

Your Workflow

1
Open the file

Open runtime_worksheet.csv in Excel or Google Sheets. You should see clip_id in column A through description in column K. If it looks like one giant column, use Data > Text to Columns (Excel) or re-import with comma delimiter.

2
Sort by chapter

Click on column B (chapter) and sort ascending. This groups everything by chapter so you can work through the story in order.

3
Scan each chapter

Read through the descriptions. You don’t need to watch any footage — just use your memory of the script and gameplay. Ask yourself: does this default feel about right?

4
Adjust where needed

Change est_seconds for any row where the default feels wrong:

  • Big moments deserve more time. War declarations, key alliances, dramatic reversals — bump these up.
  • Quick flashes need less. Menu screens, brief UI pops, minor notifications — bring to 2-3 seconds.
  • Short interviews: if a line is just “Commerce equals friendship. Right?” then 3 seconds is plenty.
  • Long interviews: descriptions are truncated at 80 chars, so word counts underestimate longer monologues. If you remember a section being 30-40 seconds, set it there.
  • Cut entirely: set to 0 if something doesn’t need to be in the episode.
5
Save

Save the file (keep it as CSV). That’s it. Let Seth know it’s done.

What to Look For

Gameplay Clips (buffer default 4s)

Most of these are quick moments — settling a town, meeting a leader, a menu click. 4 seconds is reasonable for most. Focus on the ones that should be longer:

  • Battle sequences that unfold over several seconds
  • Diplomatic confrontations you want the viewer to absorb
  • Funny or surprising moments that need a beat

And the ones that should be shorter or zero:

  • Duplicate information (two clips showing the same thing)
  • Setup actions the viewer doesn’t need to see

Gameplay Clips (40% of raw range)

These are the big sequences — full battles, exploration montages, extended gameplay. 40% is aggressive trimming. Think about:

  • Is this a highlight reel moment? Keep 40% or less.
  • Is this the main event the viewer came to see? Bump to 50-60%.
  • Is there dead time in the middle? Keep 40% or even go lower.

Interviews

The word-count estimates work well for average-paced delivery. Adjust if:

  • You speak faster in certain sections (excited, rapid-fire commentary)
  • You speak slower (dramatic pauses, building tension)
  • The description is truncated and the actual speech is much longer

What You Don’t Need to Do

  • Don’t adjust every row. Most defaults are close enough. Focus on the 20-30 rows where your gut says “that’s wrong.”
  • Don’t worry about being exact. If you think a clip should be “around 6 seconds,” just type 6. We’re estimating, not editing.
  • Don’t change any column except est_seconds. Everything else is reference data.
  • Don’t worry about the group header rows (est_seconds = 0). Those are structural markers.
  • Don’t add or delete rows. Just change numbers.
